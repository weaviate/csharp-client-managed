# Object Mapping

How Managed converts between C# objects and Weaviate wire format.

## Overview

The mapping system handles:
- **Type conversion**: C# types ↔ Weaviate DataTypes
- **Vector extraction/injection**: `float[]` properties ↔ named vectors
- **Reference resolution**: Object/Guid references ↔ Weaviate references
- **Nested objects**: Complex types ↔ Weaviate objects
- **Metadata injection**: Query metadata → `[MetadataProperty]` fields

---

## Type Conversion

### Automatic Type Inference

When using `[Property]` without explicit `DataType`, the type is inferred:

| C# Type | Weaviate DataType | Notes |
|---------|------------------|-------|
| `string` | `Text` | |
| `int` | `Int` | |
| `long` | `Int` | |
| `float` | `Number` | |
| `double` | `Number` | |
| `decimal` | `Number` | |
| `bool` | `Boolean` | |
| `DateTime` | `Date` | Converted to ISO 8601 |
| `DateTimeOffset` | `Date` | Converted to ISO 8601 |
| `Guid` | `Uuid` | |
| `byte[]` | `Blob` | Base64 encoded |
| `string[]` | `TextArray` | |
| `List<string>` | `TextArray` | |
| `int[]` | `IntArray` | |
| `List<int>` | `IntArray` | |
| `double[]` | `NumberArray` | |
| `bool[]` | `BoolArray` | |
| `DateTime[]` | `DateArray` | |
| `Guid[]` | `UuidArray` | |
| Custom class | `Object` | Nested object |
| `List<CustomClass>` | `ObjectArray` | Nested array |

### Explicit Type Specification

Override inference when needed:

```csharp
// Phone number (special string format)
[Property(DataType.PhoneNumber)]
public string Phone { get; set; }

// Geo coordinates
[Property(DataType.GeoCoordinate)]
public GeoCoordinate Location { get; set; }

// Force specific type
[Property(DataType.Text)]
public object DynamicValue { get; set; }
```

### Nullable Handling

Nullable types are supported:

```csharp
[Property]
public string? OptionalTitle { get; set; }  // Can be null

[Property]
public int? OptionalCount { get; set; }     // Can be null
```

- `null` values are stored as null in Weaviate
- Use `[Index(IndexNullState = true)]` at class level to filter by null

## Vector Mapping

### How It Works

1. **On Insert**: Vectors are extracted from `float[]` properties and sent separately
2. **On Query**: If `.WithVectors()` is used, vectors are injected back into properties

### Single Vector

```csharp
[Vector<Vectorizer.Text2VecOpenAI>]
public float[]? Embedding { get; set; }
```

- Property type: `float[]` or `float[]?`
- Stored as named vector (name = property name in camelCase)
- Auto-generated by vectorizer or user-provided

### Multi-Vector (ColBERT)

```csharp
[Vector<Vectorizer.SelfProvided>]
[Encoding(KSim = 3)]
public float[,]? TokenEmbeddings { get; set; }
```

- Property type: `float[,]` (2D array)
- Each row is a token embedding
- Used for late interaction models (ColBERT)

### Named Vector Mapping

Property name becomes vector name:

```csharp
public class Product
{
    [Vector<Vectorizer.Text2VecOpenAI>]
    public float[]? NameEmbedding { get; set; }     // → "nameEmbedding"

    [Vector<Vectorizer.Multi2VecClip>]
    public float[]? ImageEmbedding { get; set; }    // → "imageEmbedding"
}
```

Custom name:

```csharp
[Vector<Vectorizer.Text2VecOpenAI>(Name = "content_vector")]
public float[]? Embedding { get; set; }  // → "content_vector"
```

### User-Provided Vectors

For `Vectorizer.SelfProvided`, you must provide the vector:

```csharp
[Vector<Vectorizer.SelfProvided>]
public float[]? CustomEmbedding { get; set; }

// On insert
await collection.Insert(new Product
{
    Name = "Widget",
    CustomEmbedding = myEmbeddingModel.Encode("Widget")
});
```

### Retrieving Vectors

Vectors are not returned by default. Use `.WithVectors()`:

```csharp
var results = await products.Query
    .WithVectors(p => p.NameEmbedding)
    .Execute();

foreach (var product in results)
{
    if (product.NameEmbedding != null)
    {
        Console.WriteLine($"Vector dimensions: {product.NameEmbedding.Length}");
    }
}
```

## Reference Mapping

### Reference Types

Three patterns for references:

#### 1. Full Object Reference

```csharp
[Reference]
public Category? Category { get; set; }
```

- Target collection inferred from property type (`Category`)
- On query with `.WithReferences()`, full object is populated
- Type must have `[WeaviateCollection]` attribute

#### 2. ID-Only Reference

```csharp
// Guid? carries no type info — Target= is required
[Reference(Target = typeof(Author))]
public Guid? AuthorId { get; set; }
```

- Stores only the UUID
- Lightweight, no object expansion
- Use when you only need the ID

#### 3. Multi-Reference

```csharp
[Reference]
public List<Tag>? Tags { get; set; }
```

- Multiple references to same collection
- Returns list of objects when expanded

### Reference Resolution

References are resolved based on context:

**On Insert:**
```csharp
// If object has ID, uses that ID
await articles.Insert(new Article
{
    Title = "Test",
    Category = existingCategory  // Uses existingCategory's ID
});

// For new referenced objects, insert separately first
var categoryId = await categories.Insert(newCategory);
```

**On Query:**
```csharp
// Without WithReferences: Reference properties are null
var articles = await collection.Query().Execute();
// article.Category == null

// With WithReferences: Populated with data
var articles = await collection.Query()
    .WithReferences(a => a.Category)
    .Execute();
// article.Category.Name == "Technology"
```

### Current Limitations

References in Managed are **ID-based**, not navigation properties:

```csharp
// ❌ NOT SUPPORTED: Navigating through reference
article.Category.Articles  // Won't work

// ✅ SUPPORTED: Separate queries
var category = article.Category;
var articlesInCategory = await articles.Query
    .Where(a => a.CategoryId == category.Id)
    .Execute();
```

## Nested Object Mapping

### Automatic Type Inference

Nested types are inferred from property type:

```csharp
[Property(DataType.Object)]
public Address Address { get; set; }  // Infers Address type

[Property(DataType.ObjectArray)]
public List<PhoneNumber> Phones { get; set; }  // Infers PhoneNumber type
```

### Nested Type Definition

Nested types don't need `[WeaviateCollection]`:

```csharp
public class Address
{
    [Property]
    public string Street { get; set; } = "";

    [Property]
    public string City { get; set; } = "";

    [Property]
    public string PostalCode { get; set; } = "";
}
```

### Deep Nesting

Nesting can be multiple levels deep:

```csharp
public class Order
{
    [Property(DataType.ObjectArray)]
    public List<LineItem> Items { get; set; } = new();
}

public class LineItem
{
    [Property]
    public string ProductName { get; set; } = "";

    [Property(DataType.Object)]
    public Money Price { get; set; } = new();
}

public class Money
{
    [Property]
    public decimal Amount { get; set; }

    [Property]
    public string Currency { get; set; } = "USD";
}
```

### NestedType Attribute

Only needed for polymorphic scenarios:

```csharp
// Interface property - must specify concrete type
[Property(DataType.Object)]
[NestedType(typeof(EmailContact))]
public IContact PrimaryContact { get; set; }

// Base class - must specify concrete type
[Property(DataType.ObjectArray)]
[NestedType(typeof(PhoneContact))]
public List<ContactBase> Contacts { get; set; }
```

## Metadata Injection

### MetadataProperty Attribute

Query metadata can be injected into properties:

```csharp
public class SearchResult
{
    [Property]
    public string Title { get; set; } = "";

    [MetadataProperty(MetadataType.Score)]
    public float? SearchScore { get; set; }

    [MetadataProperty(MetadataType.Distance)]
    public float? VectorDistance { get; set; }

    [MetadataProperty(MetadataType.Certainty)]
    public float? Certainty { get; set; }

    [MetadataProperty(MetadataType.CreationTime)]
    public DateTime? CreatedAt { get; set; }

    [MetadataProperty(MetadataType.UpdateTime)]
    public DateTime? UpdatedAt { get; set; }
}
```

### Using Metadata

```csharp
var results = await collection.Query
    .Hybrid("search query")
    .WithMetadata(MetadataQuery.Score | MetadataQuery.Distance)
    .Execute();

foreach (var result in results)
{
    Console.WriteLine($"{result.Title}");
    Console.WriteLine($"  Score: {result.SearchScore}");
    Console.WriteLine($"  Distance: {result.VectorDistance}");
}
```

### Metadata Types

| MetadataType | Property Type | Description |
|--------------|---------------|-------------|
| `Score` | `float?` | BM25/hybrid relevance score |
| `Distance` | `float?` | Vector distance (lower = closer) |
| `Certainty` | `float?` | 1 - normalized distance |
| `CreationTime` | `DateTime?` | Object creation timestamp |
| `UpdateTime` | `DateTime?` | Last update timestamp |
| `ExplainScore` | `string?` | Detailed score explanation |

## Property Name Conversion

### Default: PascalCase → camelCase

C# convention (PascalCase) is converted to Weaviate convention (camelCase):

```csharp
public string FirstName { get; set; }   // → "firstName"
public string XMLParser { get; set; }   // → "xmlParser"
public string IOStream { get; set; }    // → "ioStream"
```

### Custom Names

Override with `Name` property:

```csharp
[Property(Name = "first_name")]
public string FirstName { get; set; }  // → "first_name"

[Vector<Vectorizer.Text2VecOpenAI>(Name = "content_embedding")]
public float[]? Embedding { get; set; }  // → "content_embedding"
```

### Collection Names

Collection name defaults to class name, override with attribute:

```csharp
[WeaviateCollection("user_profiles")]
public class UserProfile { }  // → "user_profiles"
```

## Mapping Flow

### Insert Flow

```
C# Object
    │
    ├─► PropertyMapper
    │   - Convert properties to Weaviate format
    │   - Apply name conversion
    │   - Handle nested objects
    │
    ├─► VectorMapper.Extract()
    │   - Find [Vector<T>] properties
    │   - Extract float[] values
    │   - Remove from properties
    │
    └─► ReferenceMapper.Extract()
        - Find [Reference] properties
        - Extract GUIDs
        - Build reference structure
    │
    ▼
WeaviateObject { Properties, Vectors, References }
```

### Query Flow

```
WeaviateObject { Properties, Vectors, References, Metadata }
    │
    ├─► PropertyMapper
    │   - Convert properties to C# types
    │   - Apply name conversion (reverse)
    │   - Reconstruct nested objects
    │
    ├─► VectorMapper.Inject()
    │   - Find [Vector<T>] properties
    │   - Inject vectors from response
    │
    ├─► ReferenceMapper.Inject()
    │   - Find [Reference] properties
    │   - Populate from expanded references
    │
    └─► MetadataInjector.Inject()
        - Find [MetadataProperty] properties
        - Inject score, distance, etc.
    │
    ▼
C# Object (fully populated)
```
